## 🤔 چرا این دوره وجود دارد؟

خیلی از دوستانی که وارد دنیای فناوری می‌شوند، در هر نقش و جایگاهی که هستند –  
**دیتا ساینتیست**، **مدیر محصول**، **تازه‌وارد بک‌اند** یا حتی **فرانت‌اند** –  
به یک درک مشترک از «چطور یک وب‌سایت واقعاً کار می‌کند؟» نیاز دارند.

این دوره **قرار نیست شما را یک برنامه‌نویس حرفه‌ای کند**.  
هدف این است:  
✅ با مفاهیم پایهٔ وب مثل HTTP، کلاینت، سرور، دیتابیس و لاگین آشنا شوید.  
✅ بتوانید مستندات فنی تیم را بهتر بفهمید.  
✅ دیگر از اصطلاحات برنامه‌نویسی وب نترسید.  
✅ اگر می‌خواهید مسیر بک‌اند را شروع کنید، یک نقشه‌ی ذهنی درست داشته باشید.

---

## 👥 این دوره برای چه کسانی است؟

| نقش | چرا مفید است؟ |
|------|----------------|
| 📊 دیتا ساینتیست | بفهمید داده‌ها چطور از مرورگر به API و دیتابیس می‌رسند تا بتوانید پایپلاین بهتری طراحی کنید. |
| 📋 مدیر محصول | درک کنید چرا بعضی ویژگی‌ها پیچیده‌تر از چیزی هستند که به نظر می‌رسند و چطور با تیم فنی گفتگوی مؤثرتری داشته باشید. |
| 💻 تازه‌وارد بک‌اند | یک دید درست از اولویت‌ها: HTTP، سرور، routing، لاگین و امنیت اولیه. |
| 🎨 فرانت‌اند (تازه‌کار) | ببینید درخواست‌های شما به سرور چه سرنوشتی پیدا می‌کند. |

## 📚 سرفصل‌ها (داستان وب)

1. **هفته اول** – وب چطور کار می‌کند؟ (HTTP، درخواست و پاسخ)
2. **هفته دوم** – مرورگر و سرور؛ داستان درخواست POST و GET
3. **هفته سوم** – داده‌ها کجا زندگی می‌کنند؟ (آشنایی با دیتابیس)
4. **هفته چهارم** – کوکی، سشن، احراز هویت ساده
5. **هفته پنجم** – API چیست و چطور با آن حرف می‌زنیم؟
6. **هفته ششم** – یک سناریوی واقعی: از لاگین تا نمایش پروفایل
